## 二、提示词的处理

### （二）事件-动作解耦

在一些互动型世界模型中，可将提示词解耦为事件描述和动作描述两部分。事件描述由LLM在初始生成或事件被触发的那一刻一次性处理，后续各token均可直接复用；对于动作描述，由于动作可能会出现在每一帧，故可将高频动作的embedding和对应提示词或键盘输入等的关系直接缓存。理想状态下，在推理的每一帧，模型只需将缓存好的动作Embedding与初始可复用的事件Embedding拼接，就可以直接计算下一帧，省去了LLM的实时重复计算，降低了推理延迟。

传统的串行生成过程是： 去噪预测第 N 块的潜变量 + VAE 解码第 N 块为 RGB 像素 + 去噪预测第 N + 1 块的潜变量 -+ VAE 解码第 N + 1 块为 RGB 像素 “ ． 这种方式在流式传输中， 如果去噪和解码的总时间超过了视频播放时间， 画面就会卡顿。 并行流水线 (Pipeline parallelism) 的巧妙之处在于： 自回归视频扩散模型是在潜空间中运行的。 也就是说， 预测第 N + 1 块只需要第 N 块的潜变量缓存 （KV Cache)， 根本不需要第 N 块的 RGB 像素画面。 工作流： 当模型完成第 N 块的潜变量去噪后， 它就把这个潜变量 " 分发 “ 给两条线。 线程 A (VAE 解码器）： 立刻开始把第 N 块的潜变量解码成用户能看到的 RGB 视频画面线程 B （扩散生成器）： 无需等待 VAE 解码完毕， 直接把第 N 块的潜变量作为自回归的上下文， 开始预测第 N + 1 块的潜变量。 这样， 去噪和解码在时间上重叠了， 单块处理的延迟从 “ 两者之和 " 降到了 " 两者中的最大值 “， 极大地提升了系统吞吐量， 同时完美遵守了潜空间中的块间自回归因果性。
